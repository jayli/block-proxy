# Android Client Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an Android tunnel client that establishes 1-2 TLS tunnels to a block-proxy server and lets the server reverse-connect into the Android device's local network.

**Architecture:** A Kotlin/Compose app stores configuration, obtains Android VPN user consent, and starts a foreground `VpnService`. The service establishes a minimal VPN session, protects its tunnel sockets from the VPN, and runs a coroutine-based tunnel client with per-connection frame extraction, send queues, reconnection, and reverse CONNECT relay sessions.

**Tech Stack:** Kotlin, Android Gradle Plugin, Jetpack Compose, Coroutines, DataStore, Android Keystore or AndroidX Security Crypto, JUnit, kotlinx-coroutines-test, Android instrumentation tests.

---

## Global Constraints

Every task must honor the Android design spec at `docs/superpowers/specs/2026-07-06-android-client-design.md`.

- All Android code lives under `android-client/`.
- Package name: `com.blockproxy.android` unless explicitly changed before implementation.
- Minimum SDK: 26.
- Target SDK: use the stable Android SDK generated by the Android Studio template in Task 1; if targeting Android 14+, declare foreground service type and permission.
- Tunnel server default port: `8003`.
- Initial tunnel connect timeout: 10 seconds.
- Reverse target connect timeout: 30 seconds.
- Full-frame idle timeout: 60 seconds; partial receives do not reset it.
- Reconnect backoff: 1s, 2s, 4s, cap 60s.
- Max tunnel connections: 2.
- Max frame payload: 65535 bytes; max DATA payload: 65532 bytes.
- TLS minimum: TLS 1.2.
- Default `allowInsecure = true`, scoped only to tunnel sockets.
- Reverse reqid range: `0x0001..0x7FFF`; forward reqid range reserved and unused.
- Every reverse reqid is bound to the exact `TunnelConnection` that received CONNECT.
- Outbound frames on a tunnel connection must pass through that connection's `SendQueue`.
- `CONNECT_OK` must be enqueued before target-to-tunnel DATA relay starts.
- `VpnService.protect(socket)` must be called before connecting tunnel sockets.
- Passwords must not be stored in plaintext DataStore.
- Do not add a default VPN route in v1.
- Known frame types are decoded strictly; unknown frame types are represented as `Frame.Unknown` and ignored after authentication.
- Replenishment after dual-connection degradation waits 1s, tries at most 3 times, waits 2s then 4s between failed attempts, and keeps single-connection mode if all attempts fail.
- Blocking socket reads must be interrupted by closing the socket on idle timeout; do not assume coroutine cancellation alone stops `InputStream.read()`.
- The app must remain resident in memory when the screen is off. Request battery optimization exemption via `ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` at first VPN start.
- Acquire `PARTIAL_WAKE_LOCK` while the VPN service is running to prevent CPU deep sleep.
- Use `START_STICKY` from `onStartCommand` so the system restarts the service after being killed.
- Register a periodic `WorkManager` watchdog (every 15 minutes) as a last-resort recovery mechanism.

- [ ] **Step 2: Implement frame model**

Use a sealed class:
```kotlin
sealed class Frame {
    data class Connect(val reqid: Int, val address: FrameAddress, val port: Int) : Frame()
    class Data(val reqid: Int, val payload: ByteArray) : Frame() {
        override fun equals(other: Any?): Boolean =
            other is Data && reqid == other.reqid && payload.contentEquals(other.payload)
        override fun hashCode(): Int = 31 * reqid + payload.contentHashCode()
    }
    data class Close(val reqid: Int) : Frame()
    data class ConnectOk(val reqid: Int) : Frame()
    data class ConnectFailed(val reqid: Int) : Frame()
    data object Ping : Frame()
    data object Pong : Frame()
    data class Auth(val username: String, val password: String) : Frame()
    data object AuthOk : Frame()
    data object AuthFail : Frame()
    data class Error(val message: String) : Frame()
    class Unknown(val type: Int, val payload: ByteArray) : Frame() {
        override fun equals(other: Any?): Boolean =
            other is Unknown && type == other.type && payload.contentEquals(other.payload)
        override fun hashCode(): Int = 31 * type + payload.contentHashCode()
    }
}
```

```kotlin
class TunnelWatchdogWorker(
    context: Context,
    params: WorkerParameters,
) : CoroutineWorker(context, params) {
    override suspend fun doWork(): Result {
        // 1. Read config from ConfigRepository
        // 2. Read last persisted status from StatusStore
        // 3. If config exists and last status was connected/reconnecting:
        //    - Check if BlockProxyVpnService is running
        //    - If not, start it via ContextCompat.startForegroundService()
        return Result.success()
    }
}
```

- [ ] **Step 3: Register periodic work**

In `BlockProxyVpnService.onCreate()` or `TunnelServiceController`, register the periodic worker:

```kotlin
val request = PeriodicWorkRequestBuilder<TunnelWatchdogWorker>(

).build()
WorkManager.getInstance(applicationContext).enqueueUniquePeriodicWork(

    ExistingPeriodicWorkPolicy.KEEP,
    request
)
```
