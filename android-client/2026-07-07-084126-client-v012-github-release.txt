⏺ 两份文档都已审阅完毕。以下是你在 macOS 上开发调试 Android
  客户端需要准备的环境：

  ---
  必装软件

  1. Android Studio（必须）

  这是最核心的依赖，文档明确要求用 Android Studio stable channel
  生成项目模板（Task 1 Step 1）：

  brew install --cask android-studio

  安装后首次打开会引导安装 Android SDK。选择 Empty Activity (Jetpack Compose)
  模板，Package name 设为 com.blockproxy.android，Minimum SDK 选 26。

  2. Android SDK & Platform Tools

  Android Studio 首次启动会自动下载，但需确认：

  - SDK 26+（API Level 26，Android 8.0）— minSdk
  - 最新 stable compileSdk/targetSdk — 模板生成时自动选择
  - Android SDK Platform-Tools（提供 adb）
  - Android Emulator（如果没有真机）

  3. JDK（Android Studio 自带）

  Android Studio 自带 JetBrains Runtime，一般不需要额外安装。如需单独装：

  brew install openjdk@17

  4. Gradle（不需要单独装）

  项目使用 Gradle Wrapper（gradlew），首次构建会自动下载对应版本。

  ---
  调试设备

  文档对真机测试有强要求（Task 13），包括锁屏保活、VPN 授权、WakeLock
  验证等。两种方式：

  ┌─────────────────┬──────────────────────────┬────────────────────────────┐
  │      方式       │         适用场景         │            准备            │
  ├─────────────────┼──────────────────────────┼────────────────────────────┤
  │ Android         │ 保活、VPN、厂商 ROM 验证 │ USB 线 + 开启开发者模式 +  │
  │ 真机（推荐）    │                          │ USB 调试                   │
  ├─────────────────┼──────────────────────────┼────────────────────────────┤
  │ Android         │ 单元测试、协议验证、UI   │ Android Studio AVD Manager │
  │ Emulator        │ 开发                     │  创建，建议 API 33+        │
  └─────────────────┴──────────────────────────┴────────────────────────────┘

  真机准备：
  1. 设置 → 关于手机 → 连续点击"版本号" 7 次 → 开启开发者模式
  2. 设置 → 开发者选项 → 开启 USB 调试
  3. 连接 USB，macOS 上 adb devices 应能看到设备

  ---
  验证当前环境是否就绪

  安装完 Android Studio 后，运行以下命令确认：

  # 确认 adb 可用
  adb version

  # 连接设备后确认
  adb devices

  # 进入项目目录试构建
  cd android-client
  ./gradlew :app:assembleDebug

  ---
  文档审阅意见

  两份文档质量很高，结构清晰，与 iOS 客户端保持了对齐。几点观察：

  1. 协议兼容性有保障 — 帧格式、编解码向量、双连接模型完全参照
  tunnel/protocol.js，且明确了未知帧的兼容策略（Frame.Unknown），与 Python
  客户端一致。
  2. 保活四层防御设计合理 — 电池优化豁免 + WakeLock + START_STICKY + WorkManager
  看门狗，覆盖了从标准 Android 到厂商 ROM 的各种情况。文档也务实地承认厂商 ROM
  可能绕过，需要用户手动加自启动白名单。
  3. VpnService 仅用于生命周期承载 —
  不接管设备流量、不加默认路由，这是一个关键且正确的设计决策，避免了 VPN
  对正常网络的影响。
  4. 测试策略完备 — 13 个 Task 全部采用 TDD，有单元测试 + 仪器测试 + 集成测试 +
  真机验证，覆盖了帧编解码边界、并发、重连、权限流程等。
  5. 一个注意点：minSdk = 26（Android 8.0），这意味着不支持 Android 7.x
  及更早设备。对个人部署场景这通常是可接受的。

  ---
  总结：装 Android Studio + 准备一台 Android 真机（或模拟器）
  就够了。Gradle、JDK、SDK 都会自动处理。
